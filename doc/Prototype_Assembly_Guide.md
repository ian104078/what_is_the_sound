# Arduino 拍手互動裝置：Prototype 完整組裝與測試說明書

這份說明書將帶領您一步步完成硬體接線、環境設定、以及程式碼的燒錄與測試。請按照順序進行，以確保 Prototype 能夠順利運作。

---

## 第一步：準備工作

在開始接線之前，請先確認您手邊有以下物品，並完成 SD 卡的準備。

### 1. 硬體清單 (BOM) 確認
- **主控板**：Arduino Uno R3 (或 Nano) x1
- **執行器**：MG90S 伺服馬達 (金屬齒輪) x2
- **感測器**：HC-SR04 超音波感測器 x1
- **輸入元件**：微動開關 (帶長桿或滾輪) x1 (PRD 預留 2，但實務上機構觸發 1 顆即可)
- **音效元件**：DFPlayer Mini MP3 模組 x1 + 3W 4Ω 迷你喇叭 x1 + microSD 卡 x1
- **電源與周邊**：5V/2A 行動電源 (或穩定供電的變壓器)、1KΩ 電阻 x1、麵包板與杜邦線

### 2. SD 卡音效準備
1. 將 microSD 卡格式化為 **FAT32** 格式。
2. 準備您的拍手（或任何測試）音效 MP3 檔案。
3. 將該檔案命名為 **`0001.mp3`**，並直接放置於 SD 卡的**根目錄**（不要放入任何資料夾）。
4. > [!WARNING]
   > 如果您使用 Mac 電腦，請務必清理 SD 卡中的隱藏檔（如 `._0001.mp3`），否則 DFPlayer 可能會播放失敗或當機。

完成後將 SD 卡插入 DFPlayer 模組。

## 第二步：硬體接線指南

請在**未通電**的情況下進行以下接線。

### 1. 腳位對照表

請將各元件的訊號腳位連接至 Arduino 的對應腳位：

| 元件名稱 | 元件接腳 | Arduino 接腳 | 備註說明 |
| :--- | :--- | :--- | :--- |
| **微動開關** | COM | `GND` | 共地 |
| | NO (常開端) | `D2` | 程式已設定為 INPUT_PULLUP，平時 HIGH，按下時為 LOW |
| **超音波感測器** | Trig | `D7` | 觸發訊號輸出 |
| | Echo | `D8` | 回波訊號輸入 |
| **左手伺服馬達** | Signal (橘/黃線) | `D9` | PWM 訊號 |
| **右手伺服馬達** | Signal (橘/黃線) | `D10` | PWM 訊號 |
| **DFPlayer Mini**| RX | `D11` | **[注意]** 必須在中間**串接 1KΩ 電阻**降壓，再接到 DFPlayer RX |
| | TX | `D12` | 訊號回傳 |
| | SPK+ / SPK- | 喇叭 | 連接您的 3W 小喇叭 |

### 2. 電源隔離與共地 (極度重要！)

> [!CAUTION]
> **嚴禁**將兩顆伺服馬達與 DFPlayer 的 VCC (正極) 直接接在 Arduino 的 `5V` 腳位上。馬達啟動的瞬間大電流會導致 Arduino 電壓驟降並不斷重新開機。

**正確的電源接法：**
1. **外部電源 (5V/2A)**：準備從行動電源或變壓器引出的 5V (紅線) 與 GND (黑線)。
2. **正極分流**：將外部電源的 5V 分別接給 → **左馬達 VCC**、**右馬達 VCC**、**DFPlayer VCC**。
3. **微控制器供電**：將外部電源的 5V 同時接給 **Arduino 的 5V 腳位**（若是用 USB 給 Arduino 供電，這條可省略，但極度建議統一由外部供電電源輸入 `VIN` 或 `5V` 定壓腳位）。
4. **共地 (Common Ground)**：將外部電源的 GND、Arduino 的 GND、所有馬達的 GND、DFPlayer 的 GND、超音波的 GND、微動開關的 COM **全部接在一起**（利用麵包板的負極匯流排）。

## 第三步：軟體環境設定

1. **開啟 Arduino IDE**，確保已選擇正確的開發板 (Arduino Uno) 與 COM Port。
2. **安裝依賴函式庫**：
   - 點擊左側「Library Manager」(或者使用選單：Sketch -> Include Library -> Manage Libraries...)。
   - 搜尋並安裝 **`DFRobotDFPlayerMini`** (由 DFRobot 提供)。
3. **確認程式碼位置**：
   請確認您有兩個主要的 `.ino` 可以開啟：
   - 測試用：`test_device/test_device.ino`
   - 正式用：`clapping_device/clapping_device.ino`

---

## 第四步：初始模組測試 (使用 test_device)

在組裝完整機構前，我們先進行個別模組的測試。

1. 在 Arduino IDE 中打開 `test_device/test_device.ino`。
2. 點擊 **Upload (上傳)**。
3. 上傳完成後，打開右上角的 **Serial Monitor (序列埠監視器)**。
4. 將波特率 (Baud Rate) 設定為 **`9600`**。
5. 您會看到測試選單。請依序輸入數字並按下 Enter 進行單獨測試：
   - 輸入 `1`：手在超音波前揮動，確認距離數字有變化且合理。
   - 輸入 `2`：確認兩顆馬達會來回轉動 3 次。
   - 輸入 `3`：聽聽看喇叭是否會播放 `0001.mp3`。
   - 輸入 `4`：手動按下微動開關，畫面應顯示 `>>> TRIGGERED <<<`。
   - 輸入 `5`：不按開關，觀察馬達轉動後是否會在 200ms 後強制發出聲音 (驗證 Fallback 機制)。

> [!TIP]
> 如果上述測試有任何一項失敗，請立即回頭檢查該元件的接線，確認無誤後再進入下一步。

## 第五步：機構組裝與正式運行

模組測試全數通過後，您可以進行機構組裝，並載入正式程式。

1. **馬達歸位與機構安裝**：
   - 在未接實體機構的情況下，讓程式載入停留於 `IDLE` 狀態（馬達此時在 `ANGLE_OPEN`，即張拉位置）。
   - 斷電，將手掌機構或連桿依照完全張開的角度鎖上馬達齒輪。
   - 調整微動開關的位置：當雙手閉合時，連桿必須能**剛好確實壓下**微動開關聽到「喀」一聲，但又不能壓得太深以免馬達卡死。
2. **上傳正式程式**：
   - 打開 `clapping_device/clapping_device.ino`。
   - 點擊 **Upload (上傳)**。
3. **驗收測試**：
   - **待機**：人遠離感測器 (>100cm)，裝置應靜止無聲。
   - **動態互動**：人走近 (10cm ~ 100cm)，裝置開始拍手。距離越近，拍手頻率越快，聲音越大。
   - **Fallback 驗證**：故意將微動開關的線拔掉一條。此時裝置作動時依然會發出聲音（軟體強迫同步），代表 Fallback 容錯機制運作正常。
   - **休眠**：離開感測範圍超過 5 秒，裝置自動停止拍手並回到張開狀態。

## 附錄：常見問題排除 (FAQ)

- **馬達會不規則抽搐，或 Arduino 一直重啟？**
  👉 100% 是電源問題。請檢查是否做好了電源分流，確保馬達的電流不是直接從 Arduino 抽取的。
- **DFPlayer 有底噪或雜流聲？**
  👉 檢查 Arduino D11 接到 DFPlayer RX 的中間，有沒有確實串接 1KΩ 的電阻。
- **超音波距離一直顯示 0？**
  👉 檢查 Trig 和 Echo 是否接反了，或是前面障礙物太遠/材質會嚴重吸音。
- **想要調整拍手的快慢或開合角度？**
  👉 打開 `clapping_device/config.h`，您可以自由修改 `SERVO_LEFT_OPEN`、`CLAP_DELAY_MIN_MS` 等常數，修改後重新上傳即可生效。
